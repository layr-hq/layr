# Visual Hierarchy

## What
Visual Hierarchy is the arrangement of elements in a way that guides the user’s attention in a clear order of importance.

Users should immediately understand:
- what is most important
- what to look at first
- what to do next

## Why it matters
Without clear hierarchy, everything competes for attention.

This leads to:
- confusion
- slower decision-making
- missed actions
- reduced usability

Strong hierarchy creates:
- clear focus
- faster scanning
- confident interaction

## When to apply
- all screens and layouts
- landing pages
- dashboards
- forms and flows
- any interface with multiple elements

## How to apply

- define one primary focal point
- use size, spacing, and weight to indicate importance
- reduce emphasis on secondary and tertiary elements
- guide the eye from top to bottom or left to right
- ensure the primary action stands out immediately

Users should know where to look without thinking.

## Implementation rules

- one dominant primary element per screen
- use size to indicate importance (larger = more important)
- use spacing to separate and group content
- use contrast to highlight key actions
- secondary elements must have lower visual weight
- avoid equal emphasis across multiple elements
- maintain consistent hierarchy across screens

## Example

Bad:
- all elements similar size and style
- multiple competing focal points
- unclear reading or interaction flow

Good:
- one clear primary focus
- supporting elements visually reduced
- logical flow guiding the user’s attention

## Fail conditions

- user does not know where to look
- user scans multiple areas before acting
- important elements are missed
- no clear focal point

## Enforcement rule

If everything looks important, nothing is - reduce and restructure until one clear focal point guides the user.
