# Doherty Threshold

## What it is
Users stay engaged when system response time is under ~400ms.

## Why it matters
Slow feedback breaks flow and trust.

## How to apply
- respond instantly where possible
- show loading states immediately
- avoid delays without feedback

## Example
Bad:
Click → nothing happens

Good:
Click → instant feedback or loading indicator

## Fail signs
- user clicks multiple times
- user thinks the system is broken

## Rule
Always respond immediately, even if still processing.
