# Cognitive Load Theory

## What
Cognitive Load Theory states that users have a limited capacity for processing information.  
The more mental effort required, the worse the experience.

## Why it matters
High cognitive load slows users down, increases errors, and leads to abandonment.

When users have to:
- think too much
- interpret unclear information
- make unnecessary decisions

they lose confidence and disengage.

Reducing cognitive load makes interactions feel faster, clearer, and easier.

## When to apply
- onboarding flows
- forms and data entry
- dashboards with multiple elements
- multi-step processes
- any screen requiring decisions or interpretation

## How to apply

- reduce the number of visible elements
- eliminate unnecessary decisions
- break complex tasks into simple steps
- use familiar patterns and language
- guide users instead of requiring them to figure things out

Design should remove thinking wherever possible.

## Implementation rules

- one primary action per screen
- limit visible choices to 3–5 maximum
- avoid dense text blocks or long paragraphs
- group related information visually (Gestalt)
- use clear, direct labels (no ambiguity)
- avoid requiring users to interpret meaning or guess next steps
- progressively reveal complexity only when needed

## Example

Bad:
- cluttered interface with multiple competing elements
- unclear labels requiring interpretation
- user must decide what to do next

Good:
- simple layout with clear hierarchy
- one obvious next step
- guided progression through the flow

## Fail conditions

- user hesitates before acting
- user re-reads content
- user scans multiple areas looking for direction
- user feels overwhelmed or confused

## Enforcement rule

If the user has to think, interpret, or search for what to do, reduce complexity until the next step is obvious.
