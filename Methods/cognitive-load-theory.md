# Cognitive Load Theory

## What
Users have limited mental capacity. The more they have to think, the worse the experience.

## Why it matters
High cognitive load slows decision-making, increases errors, and causes drop-off.  
Users abandon flows when they feel overwhelmed or confused.

## When to apply
- onboarding flows
- forms and inputs
- dashboards with multiple elements
- any screen requiring decisions or interpretation

## How to apply

- reduce the number of visible elements
- break complex tasks into steps
- use clear labels and familiar language
- remove unnecessary choices
- guide the user instead of asking them to decide everything

## Implementation rules

- no more than 1 primary action per screen
- limit visible options to 3–5 maximum
- avoid dense text blocks
- group related items visually
- never require user to interpret meaning

## Example

Bad:
- dense UI with multiple actions and unclear labels

Good:
- single clear action
- simple layout
- guided steps

## Fail conditions

- user hesitates
- user re-reads content
- user scans multiple areas before acting
- user feels unsure what to do

## Enforcement rule

If the user has to think, simplify until the next step is obvious.
