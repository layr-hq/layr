# Miller’s Law

## What
Miller’s Law states that users can only hold a limited number of items in working memory at once, typically around 7 plus or minus 2.

In practice, users handle far fewer items effectively in UI contexts.

## Why it matters
Presenting too much information at once overwhelms users, increases cognitive load, and slows decision-making.

When limits are exceeded, users:
- forget options
- make errors
- feel overwhelmed
- disengage

Managing information load improves clarity, speed, and usability.

## When to apply
- menus and navigation
- option lists and choices
- dashboards with multiple elements
- forms and selection inputs
- any grouped set of information

## How to apply

- limit the number of visible items in a group
- break large sets into smaller, structured groups
- use hierarchy to organise information
- progressively reveal additional options
- prioritise the most important items first

Information should be easy to scan and process at a glance.

## Implementation rules

- limit visible options to 3–5 per group where possible
- never present more than 7 items without grouping
- use categories to break larger lists into sections
- apply clear visual hierarchy to guide scanning
- avoid long, unstructured lists
- ensure users do not need to remember information across steps

## Example

Bad:
- long list of 10–15 options with no grouping
- all items presented equally
- user must scan and remember multiple options

Good:
- grouped options into smaller categories
- limited visible items per group
- clear hierarchy guiding attention

## Fail conditions

- user feels overwhelmed by the number of options
- user struggles to process or compare items
- user forgets previously seen options
- slow or delayed decision-making

## Enforcement rule

If users must process or remember too many items at once, reduce and group the information until it is easy to scan and understand instantly.
