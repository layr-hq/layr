# Signal to Noise Ratio

## What
Signal to Noise Ratio is the balance between useful, actionable information (signal) and unnecessary or distracting elements (noise).

High signal means clarity and focus.  
High noise means clutter and confusion.

## Why it matters
Too much noise hides what actually matters.

When noise is high, users:
- struggle to find important information
- lose focus on the primary action
- feel overwhelmed and distracted

Reducing noise increases clarity, speed, and usability.

## When to apply
- all screens and layouts
- dashboards with multiple elements
- content-heavy pages
- interfaces with many features or components
- any screen where focus is critical

## How to apply

- remove elements that do not support the user’s goal
- reduce visual clutter (unnecessary text, icons, decorations)
- use contrast and hierarchy to highlight important elements
- prioritise content based on importance
- simplify layout and structure

Everything on the screen should contribute to user action.

## Implementation rules

- every element must have a clear purpose
- remove decorative elements that do not guide or inform
- limit competing visual elements
- use whitespace to separate and prioritise content
- ensure primary action is visually dominant
- avoid dense layouts with too many components

## Example

Bad:
- cluttered interface with multiple competing elements
- excessive use of colors, icons, and text
- no clear focal point

Good:
- clean layout with clear hierarchy
- minimal elements focused on the task
- strong visual emphasis on primary action

## Fail conditions

- user does not know where to look
- important elements are hard to find
- interface feels overwhelming or busy
- multiple elements compete for attention

## Enforcement rule

If an element does not clearly support the user’s goal or next action, remove it until only meaningful content remains.
