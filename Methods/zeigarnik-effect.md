# Zeigarnik Effect

## What
The Zeigarnik Effect is the tendency for users to remember and feel compelled to complete unfinished tasks.

Incomplete actions create a psychological tension that drives users to continue.

## Why it matters
Unfinished tasks stay top of mind.

When used correctly, this increases:
- return rates
- completion rates
- engagement

When ignored, users:
- forget where they left off
- lose motivation
- abandon the flow

## When to apply
- onboarding flows
- multi-step processes
- incomplete forms
- dashboards with pending actions
- any task that spans multiple steps or sessions

## How to apply

- clearly show incomplete progress
- highlight unfinished tasks or steps
- make it easy to resume where the user left off
- remind users of what is left to complete
- create visible tension between incomplete and complete states

Users should feel encouraged to finish what they started.

## Implementation rules

- display incomplete states clearly (progress indicators, status labels)
- allow users to resume progress without starting over
- persist state across sessions where possible
- highlight next required action
- differentiate clearly between complete and incomplete tasks
- avoid hiding unfinished progress

## Example

Bad:
- user leaves mid-flow with no indication of progress
- no way to resume easily
- no reminder of incomplete tasks

Good:
- progress saved and visible
- clear indication of what remains
- easy continuation from last step

## Fail conditions

- user forgets unfinished tasks
- no visible indication of incomplete progress
- user cannot easily resume
- tasks abandoned without follow-up

## Enforcement rule

If a task is incomplete, it must remain visible and easy to continue until the user finishes it.
