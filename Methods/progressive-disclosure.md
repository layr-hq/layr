# Progressive Disclosure

## What
Progressive Disclosure is the practice of showing only the information and options needed at the current moment, and revealing additional detail or complexity only when required.

## Why it matters
Showing everything at once overwhelms users and increases cognitive load.

Too much information upfront leads to:
- confusion
- hesitation
- slower decisions
- abandonment

Revealing information gradually:
- reduces overwhelm
- improves focus
- increases completion rates

## When to apply
- onboarding flows
- forms and configuration screens
- dashboards with advanced features
- settings panels
- any interface with optional or complex functionality

## How to apply

- show only essential information first
- hide advanced or optional features behind secondary actions
- reveal additional detail based on user interaction or need
- guide users step by step instead of exposing everything at once
- prioritise the most important action and content

Users should only see what they need to move forward.

## Implementation rules

- primary screen must focus on one main task
- limit initial options to essential choices only
- hide advanced settings behind toggles, dropdowns, or secondary views
- reveal complexity only after the user engages or requests it
- avoid overwhelming users with full system capabilities upfront
- maintain a clear path forward at every stage

## Example

Bad:
- all settings and options visible at once
- dense interface with multiple controls
- user unsure where to start

Good:
- simple initial view with core action
- advanced options hidden until needed
- guided progression through steps

## Fail conditions

- user feels overwhelmed on first view
- too many options visible immediately
- unclear where to start
- user delays action due to complexity

## Enforcement rule

If too much information is visible at once, hide or defer non-essential elements until only what is needed to progress remains.
