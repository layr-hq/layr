# Hick’s Law

## What
The time required to make a decision increases with the number of options presented.

More choices = slower decisions.

## Why it matters
Too many options increase cognitive load, create hesitation, and reduce action.

Users faced with excessive choice will:
- delay decisions
- feel overwhelmed
- avoid acting altogether

Reducing options increases clarity, speed, and conversion.

## When to apply
- menus and navigation
- onboarding choices
- dashboards with multiple actions
- forms and configuration screens
- any screen presenting multiple options

## How to apply

- limit visible options to only what is necessary
- define one clear primary action
- reduce or remove low-value choices
- group related options together
- hide advanced or secondary options using progressive disclosure

Design should guide decisions, not present all possibilities.

## Implementation rules

- one primary action per screen
- maximum 3–5 visible options at any time
- no more than 1–2 secondary actions
- avoid equal visual weight across multiple options
- group similar actions into categories
- defer non-essential choices to later steps

## Example

Bad:
- 6 buttons with equal size and emphasis
- no clear next step
- all options shown at once

Good:
- 1 primary action clearly highlighted
- 1–2 secondary options
- additional options grouped or hidden

## Fail conditions

- user hesitates before acting
- user scans multiple options repeatedly
- unclear which option to choose
- reduced interaction or conversion

## Enforcement rule

If multiple options compete for attention, reduce them until the next action is obvious.
