# Jakob’s Law

## What
Users spend most of their time on other products, so they expect your interface to work the same way as those they already know.

Familiar patterns reduce the need to learn.

## Why it matters
Breaking expected patterns increases cognitive load, slows users down, and creates confusion.

When interfaces behave unexpectedly, users:
- hesitate
- make mistakes
- lose confidence
- abandon the task

Familiar patterns create speed, trust, and ease of use.

## When to apply
- navigation structures
- layouts and page hierarchy
- forms and inputs
- common UI components (buttons, menus, modals)
- interaction patterns (click, hover, scroll)

## How to apply

- follow established UI conventions
- place elements where users expect them
- use familiar interaction patterns
- maintain consistent behaviour across screens
- prioritise usability over originality

Design should feel intuitive without explanation.

## Implementation rules

- use standard navigation patterns (top nav, sidebar)
- keep primary actions in expected locations
- use conventional labels (e.g. “Save”, “Next”, “Cancel”)
- avoid reinventing common components
- ensure consistency across all screens and flows
- only introduce new patterns when absolutely necessary

## Example

Bad:
- custom navigation with unclear behaviour
- unconventional placement of key actions
- unfamiliar interaction patterns

Good:
- standard navigation users already understand
- predictable layout and behaviour
- familiar labels and interactions

## Fail conditions

- user does not know where to click
- user gets lost or confused
- user must learn how to use the interface
- inconsistent patterns across screens

## Enforcement rule

If a user has to learn how the interface works, replace it with familiar patterns until it feels immediately intuitive.
