# Aesthetic-Usability Effect

## What
Users perceive visually pleasing interfaces as easier to use, even if functionality is identical.

## Why it matters
First impressions heavily influence trust and usability perception.  
A clean, well-designed interface reduces perceived effort, increases confidence, and makes users more tolerant of minor issues.

Poor visual design creates immediate doubt, increases cognitive friction, and makes the product feel harder than it is.

## When to apply
- first impression screens (landing, onboarding, home)
- high-value actions (checkout, signup, primary flows)
- dashboards and complex interfaces
- any screen where trust and clarity matter

## How to apply

- maintain a clean, consistent visual system
- use spacing to create structure and breathing room
- apply clear visual hierarchy (primary, secondary, tertiary)
- ensure alignment and consistency across all elements
- use contrast to guide attention, not decoration

Visual design must support:
- clarity
- readability
- structure
- predictability

Not aesthetics for its own sake.

## Implementation rules

- consistent spacing using a defined scale (4, 8, 16, etc)
- limit font usage to 1–2 families maximum
- maintain clear hierarchy using size, weight, and spacing
- avoid clutter and dense layouts
- ensure consistent component styling across the UI
- primary actions must stand out clearly

## Example

Bad:
- inconsistent spacing and alignment
- multiple visual styles competing
- cluttered layout with no hierarchy
- visually noisy or unbalanced design

Good:
- clean layout with clear spacing
- consistent typography and components
- strong visual hierarchy
- clear focal point and balanced structure

## Fail conditions

- UI feels messy or inconsistent
- user does not trust the interface
- important elements are not visually distinct
- layout feels overwhelming or unstructured

## Enforcement rule

If it looks confusing or inconsistent, it will feel hard to use - simplify and structure until it feels clear and predictable.
