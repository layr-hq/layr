# Trust Signals

## What
Trust Signals are visual and behavioural cues that make the system feel reliable, predictable, and safe to use.

## Why it matters
Users hesitate or abandon actions if they do not trust the interface.

Lack of trust leads to:
- hesitation
- reduced interaction
- abandonment

Strong trust increases confidence and action.

## When to apply
- forms and data input
- checkout or critical actions
- onboarding
- any irreversible or important interaction

## How to apply

- maintain consistent UI and behaviour
- provide clear feedback for all actions
- confirm success and outcomes
- avoid unexpected changes or behaviour
- use clear, predictable patterns

Trust is built through consistency and clarity.

## Implementation rules

- consistent design and interaction patterns across all screens
- clear feedback for every action
- visible confirmation of completed actions
- avoid sudden layout or behaviour changes
- ensure predictable navigation and flows
- avoid hidden or unclear system states

## Example

Bad:
- inconsistent UI behaviour
- unclear system responses
- unexpected changes

Good:
- predictable interactions
- clear confirmations
- consistent patterns across screens

## Fail conditions

- user hesitates before acting
- user unsure what will happen
- inconsistent behaviour across screens
- lack of feedback or confirmation

## Enforcement rule

If the user is unsure or hesitant, increase clarity, consistency, and feedback until the interface feels predictable and safe.
