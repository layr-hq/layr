# Journey Friction

## What
Journey Friction is any element in the user’s path that slows, blocks, or complicates progress toward their goal.

Friction increases effort and reduces the likelihood of completion.

## Why it matters
Every unnecessary step, delay, or decision reduces conversion and increases drop-off.

Users expect fast, smooth progress.  
When friction is present, users:
- hesitate
- lose momentum
- abandon the flow

Reducing friction increases speed, completion, and satisfaction.

## When to apply
- onboarding flows
- sign up and login
- checkout processes
- multi-step forms
- any task with multiple actions or steps

## How to apply

- remove unnecessary steps
- reduce the number of inputs required
- eliminate redundant actions
- streamline navigation between steps
- prioritise the shortest path to value

The user should move forward with minimal resistance.

## Implementation rules

- minimise steps required to complete a task
- remove non-essential fields and inputs
- avoid forcing users to make unnecessary decisions
- reduce page transitions where possible
- allow progress without requiring full setup upfront
- ensure each step has a clear purpose

## Example

Bad:
- long forms with unnecessary fields
- multiple steps with unclear purpose
- forced decisions before progress

Good:
- minimal inputs required
- direct path to completion
- clear, guided progression

## Fail conditions

- users drop off before completion
- users abandon mid-flow
- users hesitate or delay progress
- tasks feel longer or harder than necessary

## Enforcement rule

If a step does not directly help the user reach their goal, remove it or simplify it until the path is as short and clear as possible.
