# Feedback Loops

## What it is
Systems must respond to user actions so they know what’s happening.

## Why it matters
Without feedback, users feel lost and unsure.

## How to apply
- show immediate response to actions
- display loading states
- confirm success or failure clearly

## Example
Bad:
Click → nothing happens

Good:
Click → visual response + confirmation

## Fail signs
- repeated clicks
- confusion about system state

## Rule
Every action must have a clear reaction.
