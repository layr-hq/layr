# Affordance

## What it is
Design that suggests how something should be used.

## Why it matters
Users shouldn’t have to guess how to interact.

## How to apply
- make buttons look clickable
- use familiar patterns
- provide visual cues for interaction

## Example
Bad:
Text that looks like a button but isn’t

Good:
Clear, styled button with hover/active states

## Fail signs
- user doesn’t click interactive elements
- confusion about what is clickable

## Rule
If it can be used, it should look usable.
