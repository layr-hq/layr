# Default Bias

## What
Default Bias is the tendency for users to accept pre-selected options rather than making an active choice.

Users are significantly more likely to go with whatever is already selected.

## Why it matters
Defaults strongly influence behaviour, speed up decision-making, and reduce cognitive load.

Good defaults:
- increase completion rates
- reduce friction
- guide users toward optimal outcomes

Bad defaults:
- create confusion
- lead to poor choices
- reduce trust

## When to apply
- forms and inputs
- onboarding flows
- settings and configuration screens
- pricing selections
- any decision with a recommended option

## How to apply

- pre-select the most common or beneficial option
- use defaults to remove unnecessary decisions
- ensure the default aligns with user goals
- make it easy to change, but not required to proceed
- avoid forcing users to configure everything manually

Defaults should guide, not trap.

## Implementation rules

- always provide a default where possible
- default must reflect the best outcome for most users
- optional choices should not block progress
- avoid empty states that require user input before proceeding
- ensure the default is clearly visible and understandable
- allow easy override without friction

## Example

Bad:
- no option selected
- user forced to choose before continuing
- unclear which option is best

Good:
- best option pre-selected
- user can proceed immediately
- alternative options available but secondary

## Fail conditions

- user hesitates or is unsure what to choose
- user must configure basic settings before continuing
- default does not align with user needs
- users frequently change the default due to poor selection

## Enforcement rule

Always set a clear, sensible default so the user can move forward without needing to decide.
