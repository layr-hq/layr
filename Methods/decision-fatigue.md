# Decision Fatigue

## What it is
The more decisions a user makes, the worse their decisions become over time.

## Why it matters
Too many choices leads to hesitation, poor decisions, or no action at all.

## How to apply
- reduce the number of decisions required
- set smart defaults
- guide users instead of asking them to choose everything
- group related decisions together

## Example
Bad:
User must configure everything manually

Good:
System pre-selects the best option with ability to change

## Fail signs
- user abandons flow
- user delays action
- inconsistent user choices

## Rule
Reduce decisions, especially early in the journey.
