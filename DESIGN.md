DESIGN SYSTEM
===============================

===============================

1. ROLE

You are NOT decorating UI.

You are building:
→ clear, readable, structured interfaces

===============================

2. ZERO CONTEXT RULE

User scans in seconds.

Must:
- one focal point
- instant hierarchy
- readable at glance

===============================

3. COLOR SYSTEM

Use only:
- background
- surface
- primary
- secondary
- text
- border
- state

Rules:
- primary = main action
- no decorative use
- contrast drives attention

===============================

4. TYPOGRAPHY

Use max 2 fonts

Scale:
- H1: 28–32
- H2: 22–24
- H3: 18–20
- Body: 14–16
- Small: 12–13

Rules:
- hierarchy via size + spacing

===============================

5. SPACING

Use:
4 / 8 / 12 / 16 / 24 / 32 / 40 / 48 / 64

Rules:
- padding ≥ 16
- consistent spacing
- group related items

===============================

6. HIERARCHY

Every screen:

- primary
- secondary
- tertiary

Rules:
- size + spacing define importance

===============================

7. COMPONENTS

Use:
- buttons
- inputs
- cards
- modals
- lists

States:
- default
- hover
- active
- disabled
- loading

===============================

8. ALIGNMENT

Rules:
- strict alignment
- no floating elements

===============================

9. INTERACTION

Rules:
- target ≥ 40x40
- instant response

===============================

10. ANIMATION

Use only to:
- guide attention
- explain change

Timing:
100–300ms

===============================

11. CLARITY

Remove anything that does not:
- inform
- guide
- support action

===============================

12. VALIDATION

Check:
- hierarchy
- spacing
- clarity
- consistency

Fix until:
- clean
- obvious
- readable

===============================

FINAL RULE

- clarity > style
- consistency > creativity
- function > decoration

GOAL:

User sees → understands → acts

===============================
END
